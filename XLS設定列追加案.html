XLS 設定列追加案

勤務表 XLS にユーザー記入用の設定列と月次設定セルを追加し、残っている条件付き項目を config 手編集なしで扱えるようにするための具体案です。 想定対象は [【統一書式】あかねっこ1月.xls] と同系統の帳票で、AP 列以降の空き領域を使う前提です。

更新日: 2026-04-18 前提: 1月帳票の AP 以降は空欄確認済み 目的: 条件付き項目の入力運用を XLS 側へ寄せる 推奨初期値: AV2=1 / AW2=1 / AX2=休+特+夜休
列追加向き 3

No.9, No.10, No.12 は職員行フラグと数値入力で進めやすい

列 + 月次設定向き 2

No.11, No.14 は個人値と全体定義の両方が必要

日付側入力が必要 1

No.13 は日付の意味付けを職員列だけでは表せない

読込追加点 2系統

職員設定列読込 と 月次共通設定読込 を追加する想定

基本方針

提案する配置
  • AP から AU: 職員ごとの設定列
  • AV から BA: 月次共通設定セル
  • 必要なら BB 以降: No.13 用の拡張欄

1月帳票では AP から AX 付近の上部と職員行が空欄だったため、この領域を設定専用に使う設計は現実的です。 既存の勤務記号セルとは別領域なので、日々のシフト入力と干渉しにくい利点があります。

職員設定列案

列 項目名 対象ルール 入力値 意味 備考
AP 夜勤公平化対象 No.9 ○ / 1 / TRUE / 空欄 この職員を夜勤回数平等化の比較対象に含める ユーザーがイメージしやすいので、まず最初に入れる価値が高い列です。
AQ 夜夜必須対象 No.10 ○ / 1 / TRUE / 空欄 この職員に 夜夜 を最低回数入れる対象とする 最低回数を全員共通で 1 回に固定するなら、この列だけでも運用可能です。
AR 夜夜必須回数 No.10 整数 / 空欄 AQ が有効な職員に対する 夜夜 の最低回数 空欄時は月次共通値を使う、または 1 回固定にする実装が自然です。
AS 土日休公平化対象 No.12 ○ / 1 / TRUE / 空欄 この職員を土日休み平等化の比較対象に含める 夜勤公平化対象と似た使い方なので、ユーザー説明もしやすいです。
AT 個別連勤上限 No.11 整数 / 空欄 この職員の hard 連勤上限 No.11 の意味を固めたあとに使う列です。未使用なら空欄でよいです。
AU 4連勤許容回数 No.11 整数 / 空欄 この職員に 4 連勤を月何回まで許すか 4連勤を soft で扱う方針にする場合に使います。

職員行の設定は、その職員の行に直接記入する方式を想定します。氏名は既存どおり D 列ベースで照合します。

月次共通設定セル案

セル候補 項目名 対象ルール 入力値 用途
AV2 夜勤公平化許容差 No.9 整数 夜勤公平化対象の最大差を指定します。基本は 1 を想定します。
AW2 土日休公平化許容差 No.12 整数 土日休公平化対象の最大差を指定します。
AX2 土日休カウント方式 No.12 休のみ / 休+夜休 / 休+特+夜休 何を土日休みとして数えるかを月単位で固定します。
AY2 夜休連鎖定義 No.14 夜休セル連続 / 夜勤サイクル連続 夜休が 10 日以上続く の意味を固定します。
AZ2 夜休月跨ぎ判定 No.14 含む / 含まない previous_tail を連鎖判定に含めるか指定します。
BA2 夜休連鎖閾値 No.14 整数 通常は 10 を入れます。将来の別施設流用も見据えた欄です。
推奨初期値
  • AV2 は 1、AW2 は 1、AX2 は 休+特+夜休 を初期値として入れる運用を推奨します
  • この組み合わせは現行の JSON 既定値と一致するため、XLS 側入力へ寄せても既存挙動を変えません
  • テンプレート反映済みなので、未入力のままでも月次設定欄の意図を把握しやすくなります
ヘッダ配置案
  • AP5 から AU5 に項目名を記載
  • AV1 から BA1 に 月次設定 と見出しを記載
  • AV2 から BA2 に実値、AV3 から BA3 に説明文を置くと運用しやすいです
  • 現時点の既定値は AV2=1、AW2=1、AX2=休+特+夜休 です

No.13 の扱い

通常の日 は職員属性だけでは決まらないため、職員右端列だけでは不足します。実装するなら次の 2 案のどちらかが必要です。

案A: 日付ヘッダで通常日マーク

  • 各日付の上部に 通常日 フラグ行を追加する
  • 例: 4 行目 日付、5 行目 曜日、6 行目 通常日マーク
  • 日勤記号が入っていて、かつ通常日マーク付きの日だけをカウントする

案B: 専用勤務記号を作る

  • 通常の日 用の専用記号を追加する
  • その記号を shift_kinds に追加して別種の日勤として扱う
  • コードは整理しやすいが、現場の記入ルール変更が必要です

現場負担が少ないのは案A、コードが明確になるのは案Bです。

読込仕様案

論点 提案
ブール入力の解釈 ○ / 1 / TRUE / true / yes を有効、空欄を無効として扱う
数値入力の解釈 整数のみ有効。小数や文字列は検証レポートで要注意に出す
優先順位 XLS 入力値 ＞ period_overrides ＞ 基本 config
未入力時 既存 config を使う。XLS 未入力で動かなくしない
対象行の判定 既存の氏名照合ロジックを使い、D 列氏名と aliases で解決する
異常値の扱い 即停止ではなく、まずは validation HTML に警告を出し、必要なら strict 側は既定値へフォールバック
