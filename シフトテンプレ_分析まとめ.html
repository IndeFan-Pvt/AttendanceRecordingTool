2. 履歴から確定的に見えたこと

月別 roster は固定ではない

あかねっこは毎月 12人固定ではなく、12人から14人の間で増減しています。竹内孝寛さん、丹羽瑞穂さん、笹谷正人さんは登場月が限定的で、有村瑞穂さん、坂本泰平さん、牧山泰徳さんは一部月のみでした。単一の employees 配列だけで通年の実態を表すのは弱いです。

読み取り上の表記ゆれと誤読がある

勤務記号に 3.0 と 3,0 の表記ゆれがありました。また 12月テンプレートでは 菅野実里早日 という不自然な名前が 1件出ており、行マッチングが勤務記号を巻き込んだ可能性があります。

観点 履歴で確認できた事実 示唆
夜勤実績 竹内孝寛さん、菅野実里さんは履歴上 夜 が 0 回 少なくとも現行 roster では夜勤不可候補
パート勤務記号 八重美貴さんは 5.5 に加えて 3.0 系あり。稲富茜里さんは 6.0 主体だが 3.0 系あり。石井都子さんは 6.0 と休のみ パートを一律扱いせず、職員別 allowed_shifts を分けるべき
日勤偏重 丹羽瑞穂さんは 日 比率が高く、牧山泰徳さんは ほぼ 日 と休のみ 再登場時は 日勤要員として優先配置すると実態に近い
休日数 フル職員の休は概ね月 9日から10日付近。パートは個人差が大きい 一律レンジより個人別レンジの方が履歴に合う

3. 設定だけで先に直せる候補

現行設定ファイル: akanecco_shift_config.json。ここはスクリプト本体を変えなくても反映可能です。

4. スクリプト改修として効く候補

候補 内容 効果
月別 active roster 対応 period_overrides ごとに、その月だけ有効な職員一覧を持てるようにする 履歴に近い人数構成で解けるようになり、退職や応援月の表現も可能になる
履歴から設定候補を自動生成 過去テンプレートから 夜勤可否、allowed_shifts、平均休日日数、日勤偏重などを抽出して JSON 候補を出す 毎月の設定調整が大幅に減る
勤務記号の正規化強化 3,0 を 3.0 と同一視するなど、読み込み時に記号ゆれを吸収する 履歴集計と実運用のズレが減る
社員行の妥当性チェック 名前セルに勤務記号が混じった行を除外し、不自然な候補名を警告に出す 菅野実里早日 のような誤読を早期に検知できる
日勤偏重のソフト制約 特定職員に 日 を寄せる重み付けを追加する 履歴に近い配置を出しやすくなる

5. 要確認の論点

  • 履歴に出てきた人を、そのまま現在の config に戻すべきとは限りません。異動や退職の可能性があります
  • 八重美貴さんと稲富茜里さんは土日休が多いですが、本人希望なのか業務都合なのかは履歴だけでは断定できません
  • 丹羽瑞穂さん、牧山泰徳さんの日勤偏重も、役割固定なのか当時の一時事情なのか確認が必要です

結論
履歴から見て、まず着手価値が高いのは「夜勤不可の明示」「パート短時間記号の拡張」「3.0 と 3,0 の正規化」「月別 roster の扱い」です。この4点は、実績とのズレが明確で、導入効果も大きいです。
