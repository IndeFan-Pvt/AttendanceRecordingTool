2. 履歴から確定的に見えたこと

月別 roster は固定ではない

あかねっこは毎月 12人固定ではなく、12人から14人の間で増減しています。竹内孝寛さん、丹羽瑞穂さん、笹谷正人さんは登場月が限定的で、有村瑞穂さん、坂本泰平さん、牧山泰徳さんは一部月のみでした。単一の employees 配列だけで通年の実態を表すのは弱いです。

読み取り上の表記ゆれと誤読がある

勤務記号に 3.0 と 3,0 の表記ゆれがありました。また 12月テンプレートでは 菅野実里早日 という不自然な名前が 1件出ており、行マッチングが勤務記号を巻き込んだ可能性があります。

観点 履歴で確認できた事実 示唆
夜勤実績 竹内孝寛さん、菅野実里さんは履歴上 夜 が 0 回 少なくとも現行 roster では夜勤不可候補
パート勤務記号 八重美貴さんは 5.5 に加えて 3.0 系あり。稲富茜里さんは 6.0 主体だが 3.0 系あり。石井都子さんは 6.0 と休のみ パートを一律扱いせず、職員別 allowed_shifts を分けるべき
日勤偏重 丹羽瑞穂さんは 日 比率が高く、牧山泰徳さんは ほぼ 日 と休のみ 再登場時は 日勤要員として優先配置すると実態に近い
休日数 フル職員の休は概ね月 9日から10日付近。パートは個人差が大きい 一律レンジより個人別レンジの方が履歴に合う

4. スクリプト改修として効く候補

候補 内容 効果
月別 active roster 対応 period_overrides ごとに、その月だけ有効な職員一覧を持てるようにする 履歴に近い人数構成で解けるようになり、退職や応援月の表現も可能になる
履歴から設定候補を自動生成 過去テンプレートから 夜勤可否、allowed_shifts、平均休日日数、日勤偏重などを抽出して JSON 候補を出す 毎月の設定調整が大幅に減る
勤務記号の正規化強化 3,0 を 3.0 と同一視するなど、読み込み時に記号ゆれを吸収する 履歴集計と実運用のズレが減る
社員行の妥当性チェック 名前セルに勤務記号が混じった行を除外し、不自然な候補名を警告に出す 菅野実里早日 のような誤読を早期に検知できる
日勤偏重のソフト制約 特定職員に 日 を寄せる重み付けを追加する 履歴に近い配置を出しやすくなる

5. 要確認の論点

  • 履歴に出てきた人を、そのまま現在の config に戻すべきとは限りません。異動や退職の可能性があります
  • 八重美貴さんと稲富茜里さんは土日休が多いですが、本人希望なのか業務都合なのかは履歴だけでは断定できません
  • 丹羽瑞穂さん、牧山泰徳さんの日勤偏重も、役割固定なのか当時の一時事情なのか確認が必要です

結論
履歴から見て、まず着手価値が高いのは「夜勤不可の明示」「パート短時間記号の拡張」「3.0 と 3,0 の正規化」「月別 roster の扱い」です。この4点は、実績とのズレが明確で、導入効果も大きいです。

8. 常勤だけを前提にした設定修正候補

現行 config の full 職員だけを対象に、去年の履歴実績からそのまま候補化しやすい値を整理しています。ここは即時適用ではなく、変更候補の具体値です。

対象 履歴実績 現行設定 設定修正候補
菅野実里 2か月分で 早 17、日 22、休 17、遅 0、夜 0 allowed_shifts は 早・遅・日・夜・休・特。夜 max 4 allowed_shifts を 早・日・休・特 に絞る候補が最有力です。少なくとも 夜 は外し、理事長修正版に寄せるなら 遅 も外します
竹内孝寛 4か月分で 早 34、遅 35、日 16、休 37、夜 0 allowed_shifts は 早・遅・日・夜・休・特。夜 max 4 allowed_shifts から 夜 を外す候補です。遅 は履歴実績があるため、菅野実里さんと同じ制約にはしない方が自然です
川部春香、都築良弘、嶋﨑光子、岡虹光、鈴木美奈、駒倉利光 11か月分で夜勤平均は月 3.9 から 4.3 回、休日平均は月 9.2 から 9.8 日 夜 min 3、max 4。休 6から10 夜 min 3、max 4 は概ね維持でよい候補です。休日日数は 6から10 のままでも解けますが、履歴に寄せるなら 8から10 付近への見直し候補があります
笹谷正人 1か月分のみで 早 7、遅 7、日 3、夜 4、休 10 夜 min 3、max 4。休 6から10 履歴が 1か月だけなので、現行設定を維持しつつ保留が妥当です。常勤側の修正優先度は高くありません
常勤共通 月別 active roster が変動し、常勤だけでも固定 roster ではありません employees は固定一覧 period_overrides 単位で active 職員を持てるようにする候補です。設定だけではなくスクリプト側の拡張候補でもあります

優先順位
常勤だけを前提にするなら、先に直す候補は 1. 菅野実里さんの 遅・夜 制限、2. 竹内孝寛さんの 夜 制限、3. 常勤の休日日数レンジ再調整 の順です。

9. 常勤前提での推奨着手順

非常勤を除いた比較結果を前提に、常勤側の差分解消を優先する順で並べ替えています。

順番 作業 理由
1 config で菅野実里の 遅・夜 制限を見直す 理事長修正版に最も近い常勤差分で、履歴実績も明確です
2 config で竹内孝寛の 夜 制限を見直す 夜勤実績 0 回が明確で、菅野実里とは別パターンの常勤制約として整理できます
3 常勤の休日日数レンジを履歴寄りに再調整する 主要な常勤6名の実績は月 9日台後半に集まっており、現行の 6から10 より実態に寄せられます
4 月別 active roster を持てるようにする 常勤だけでも在籍者が月ごとに変わるため、固定 roster 前提の弱さを解消できます
5 名前誤読チェックと履歴ベース候補出力を追加する 菅野実里早日 のような誤読検知と、今後の設定見直しの自動化をまとめて強化できます
